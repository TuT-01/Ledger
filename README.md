# Bking

Bking is a local-first Android accounting app for personal asset and liability management.

## Current foundation

- Android app module with Kotlin and Jetpack Compose.
- Hilt-ready application and activity entry points.
- Room, DataStore, Navigation, Lifecycle, Coroutines, Hilt, and KSP dependencies wired in.
- Double-entry accounting domain draft model.
- Account balance calculator for asset, liability, income, expense, and equity normal balances.
- Room database foundation with accounts, transactions, and ledger entries.
- Ledger repository that writes a transaction and its entries in one Room transaction.
- Unit tests covering expense, income, transfer, credit card spending, repayments, lending, borrowing, investment trades, and opening balances.
- Instrumented tests for Room persistence and repository writes.
- First Compose dashboard for total assets, liabilities, net worth, monthly cash flow, quick actions, and account groups.

## Local setup

This workspace currently needs Android tooling before it can build from the command line.

1. Install Android Studio.
2. Open this folder: `E:\Python\Bking`.
3. Let Android Studio install Android SDK Platform 36 and sync Gradle.
4. Use JDK 17 if Android Studio asks for a Gradle JDK. The system currently has Java 25 installed, which may be newer than some Gradle or Android Gradle Plugin combinations expect.

## Verification commands

Run these after Android Studio or Gradle is available:

```powershell
gradle testDebugUnitTest --tests com.bking.domain.service.DoubleEntryFactoryTest
gradle connectedDebugAndroidTest
gradle assembleDebug
```

If a Gradle wrapper is generated by Android Studio, prefer:

```powershell
.\gradlew testDebugUnitTest --tests com.bking.domain.service.DoubleEntryFactoryTest
.\gradlew connectedDebugAndroidTest
.\gradlew assembleDebug
```

If command-line Gradle uses Java 25 and fails, set Gradle JDK to Android Studio's Embedded JDK. On this machine it is installed at `D:\Android_stdio\jbr`.
